# Implementation Details

## Overview
The **flang‑ir‑tracer** uses the **FlangDriver** class (found in `src/flang_tracer/driver.py`) to invoke the `flang‑new` binary with the `-O0` optimisation level. For each Fortran source file the driver sequentially dumps the following compilation stages:

- **Parse Tree** – raw syntactic tree generated by Flang’s parser.
- **FIR** – Flang Intermediate Representation (first‑level IR).
- **HLFIR** – High‑level FIR, enriched with type information.
- **LLVM‑IR** – Final LLVM intermediate representation.
- **Memory Stats** – Collected via Flang’s `-mem-stats` flag and parsed from the driver output.

The driver parses the stdout of each stage, extracting the textual representation and the wall‑clock duration reported by Flang. The duration values are stored in the JSON structure returned by `FlangDriver.get_all_dumps()` and later consumed by the UI and benchmarking scripts.

## LLVM Details
- The driver invokes `flang‑new -c -emit-llvm <file>.f90 -o -` to obtain the LLVM‑IR dump.
- No custom LLVM passes are applied; the focus is on measurement and visualization.
- The extracted LLVM‑IR content is fed to the **IRStats** helper to compute a lightweight instruction‑count metric (count of lines containing an ` = ` assignment).

## Memory Tracking
Memory usage is gathered by adding the `-mem-stats` flag to the Flang command line. The driver extracts lines that match the pattern `/Memory used: (\d+) MB/` and stores the numeric value in the `memory_mb` field of the JSON dump.

## Extensibility
The modular design allows additional stages (e.g., optimization passes) to be added by extending `FlangDriver.get_all_dumps()` and updating the UI components in `src/flang_tracer/ui/`.
